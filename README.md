# PuzzleBeadMap

拼豆图案互通格式 — 一个用于描述熔珠/拼豆（Perler Bead）图案的开放 JSON 格式。

## 解决什么问题

不同的拼豆工具之间无法互通图案数据：转换器 A 导出的文件，施工助手 B 无法直接导入。
PuzzleBeadMap 提供一个与工具无关的标准格式，让图案数据可以自由流转。

## 核心特性

- **自包含**：单文件包含图案还原所需的全部信息，不依赖外部色卡数据库  
- **多品牌**：原生支持多品牌混搭，每颗豆保留品牌 + 色号 + RGB 值  
- **可移植**：标准 JSON，任何语言均可读写  
- **可转换**：附带 RGB 值，支持通过 ΔE 色差算法将未知品牌转换到本地色卡  
- **人类可读**：纯文本，可直接用文本编辑器查看和修改

## 最小示例

文件扩展名推荐 `.txt`（也可用 `.json`），编码 UTF-8。

```json
{
  "format": "PuzzleBeadMap",
  "version": "1.0",
  "canvas": { "cols": 3, "rows": 2 },
  "palette": [
    { "id": 0, "brand": "hama", "brandLabel": "Hama",
      "code": "H01", "name": "White", "rgb": [229, 236, 241] },
    { "id": 1, "brand": "hama", "brandLabel": "Hama",
      "code": "H10", "name": "Black", "rgb": [30, 30, 35] }
  ],
  "grid": [
    [0, 1, 0],
    [1, 0, 1]
  ]
}

grid 中的数字是 palette 的 id 引用，null 表示空白格。

## 几个写作决策说明

**为什么先写"解决什么问题"**：GitHub 首页访问者大多是其他开发者，他们第一眼想知道"我为什么需要这个"，而不是"这是什么格式定义"。

**为什么例子用 3×2 而不是复制 SPEC 里的 6×4**：越小越好，让人一眼能数清格子和调色板的对应关系，理解 palette+id 的设计意图。

**"支持此格式的工具"表格**：这是开放格式 README 的标准写法，既展示了你的小程序，也为将来其他工具留了位置，营造"生态"感。

**不需要在 README 里写校验规则**：那些细节放 SPEC.md 就好，README 的作用是让人愿意点进 SPEC.md。# puzzlebeadmap-spec
